1.1 — Requests Retry + Backoff
--------------------------------
Sorun: Ağ anlık kesilirse direkt fail, retry yok.

Çözüm — utils/http.py olarak çıkarılacak:

    import time, requests
    from requests.adapters import HTTPAdapter
    from urllib3.util.retry import Retry

    def get_session(retries=3, backoff=1.0) -> requests.Session:
        session = requests.Session()
        adapter = HTTPAdapter(max_retries=Retry(
            total=retries,
            backoff_factor=backoff,
            status_forcelist=[429, 500, 502, 503, 504]
        ))
        session.mount("https://", adapter)
        return session

Tüm requests.get() çağrıları bu session ile değiştirilecek.

2.2 — Retry Loop'u Ortak Fonksiyona Çıkar
-------------------------------------------
Şu an 5 yerde aynı pattern. Çözüm — core/quality.py:

    def post_with_retry(generator_fn, max_attempts=3, quality_threshold=6.0):
        """
        generator_fn: () -> str  (tweet text üretir)
        Returns: tweet text veya None
        """
        for attempt in range(max_attempts):
            try:
                text = generator_fn()
                if not text or len(text) < 10:
                    continue
                if tweet_archive.is_semantically_duplicate(text):
                    continue
                score = score_tweet_quality(text)
                if score >= quality_threshold:
                    return text
            except Exception as e:
                print(f"Attempt {attempt+1} failed: {e}")
        return None

3.1 — Structured Logging
--------------------------
Şu an: print("Tweet posted: ...")
Olması gereken — utils/logger.py:

    import logging, json
    from datetime import datetime, timezone

    def get_logger(name: str) -> logging.Logger:
        logger = logging.getLogger(name)
        handler = logging.StreamHandler()
        handler.setFormatter(JsonFormatter())
        logger.addHandler(handler)
        logger.setLevel(logging.INFO)
        return logger

    class JsonFormatter(logging.Formatter):
        def format(self, record):
            return json.dumps({
                "ts": datetime.now(timezone.utc).isoformat(),
                "level": record.levelname,
                "module": record.name,
                "msg": record.getMessage(),
                **(record.__dict__.get("extra", {}))
            })

GitHub Actions bu JSON logları parse edebilir, ileride herhangi bir
log aggregator'a taşınabilir.

3.4 — failed_tweets.json Otomatik Review
------------------------------------------
Sorun: failed_tweets.json büyüyor ama kimse bakmıyor.

Çözüm — analytics.py'ye eklenecek:

    def report_failures(days=7):
        try:
            with open("failed_tweets.json") as f:
                failures = json.load(f)
        except (FileNotFoundError, json.JSONDecodeError):
            return

        cutoff = now() - timedelta(days=days)
        recent = [f for f in failures if parsed(f["failed_at"]) > cutoff]

        if not recent:
            print("No failures in last 7 days.")
            return

        by_type = {}
        for f in recent:
            ct = f.get("content_type", "unknown")
            by_type[ct] = by_type.get(ct, 0) + 1

        print(f"\n=== FAILURES (last {days} days) ===")
        for ct, count in sorted(by_type.items(), key=lambda x: -x[1]):
            print(f"  {ct}: {count} failures")


3.5 — Parallel RSS Fetching
------------------------------
Sorun: community_pulse ve data_viz 12 feed'i sırayla çekiyor = 30-60s.

Çözüm — core/rss.py:

    from concurrent.futures import ThreadPoolExecutor, as_completed

    def fetch_all_feeds(feed_dict: dict, max_items=10) -> dict:
        results = {}

        def fetch_one(source, url):
            try:
                return source, _parse_rss_all(url, source, max_items)
            except Exception:
                return source, []

        with ThreadPoolExecutor(max_workers=6) as executor:
            futures = {
                executor.submit(fetch_one, src, url): src
                for src, url in feed_dict.items()
            }
            for future in as_completed(futures, timeout=30):
                source, articles = future.result()
                results[source] = articles

        return results

Beklenen iyileşme: 30–60s → 5–10s
